# CS60004-LAB3-RL

## 数据预处理

抽取 10000 条数据，其中 8000 条作为训练集，2000 条作为测试集。由于时间问题，一般仅使用前 100 条测试数据进行评测。

如无特殊说明，均使用官方推荐的推理参数（`Temperature=0.6`，`TopP=0.95`，`TopK=20`）。

Qwen3-0.6B，最大输出长度 1024，批大小 128，使用 `Transformers` 耗时 1:34 正确率 0.43 (43/100)，使用 `VLLM` 耗时 0:12 正确率 0.41 (41/200)。虽然由于 100 条数据太少，在运行中发现结果有一定随机性，但由于 VLLM 的速度优势，后续仅需推理的情况下都选择在 VLLM 上进行。

将最大输出长度增加到 2048，正确率 0.56，继续增加到 4096，正确率 0.66。可以看到输出长度的增加对正确率有显著提升。

将 `Temperature` 增加到 1.0，正确率 0.38，继续增加到 1.5，正确率 0.33。降低到 0.1，正确率 0.5，降低到 0 变为贪婪解码，正确率 0.44。尽管官方说不要使用贪婪解码，但根据实验结果，在这个任务上贪婪解码甚至能够带来一定的性能提升。


Qwen3-8B 批大小 32，最大输出长度 1024 耗时 0:41 正确率 0.41，输出长度增加到 2048 正确率 0.67，继续增加到 4096 正确率 0.8，可以发现在最大输出长度仅有 1024 的情况下，8B 模型的表现和 0.6B 模型不相上下，但是随着输出长度的增加，8B 模型的正确率提升更为显著。
